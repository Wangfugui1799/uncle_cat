**问题分析**：

* 魔鬼编辑审阅和读者反馈模拟共享了同一个 `isEditing` 状态变量

* 当用户在读者反馈模拟中点击"确认修改"时，会设置 `isEditing = true`

* 导致魔鬼编辑审阅的UI显示为"审阅中..."，给用户带来混淆

**解决方案**：

* 为每个功能模块创建独立的状态变量

* 魔鬼编辑审阅使用 `isDemonEditing` 状态

* 读者反馈模拟使用 `isFeedbackEditing` 状态

* 这样可以避免不同功能之间的状态冲突

**修改内容**：

1. **更新状态管理**：

   * 移除共享的 `isEditing` 状态

   * 添加 `isDemonEditing` 和 `isFeedbackEditing` 状态

2. **修改魔鬼编辑审阅相关函数**：

   * `handleDemonCritique`：使用 `isDemonEditing` 状态

   * `handleApplyDemonRewrite`：使用 `isDemonEditing` 状态

3. **修改读者反馈模拟相关函数**：

   * `handleUserFeedbackRewrite`：使用 `isFeedbackEditing` 状态

4. **修改UI组件**：

   * 魔鬼编辑审阅按钮：使用 `isDemonEditing` 状态

   * 读者反馈模拟按钮：使用 `isFeedbackEditing` 状态

   * 改写方案按钮：使用 `isDemonEditing` 状态

**修改位置**：

* `components/WritingStep.tsx`：主要修改文件

**预期效果**：

* 当用户在读者反馈模拟中点击"确认修改"时，魔鬼编辑审阅不会显示为"审阅中..."

* 每个功能模块只受自己的状态影响，UI表现更加准确

* 避免了不同功能之间的状态冲突
